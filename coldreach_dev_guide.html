03Phase
Email Variants · A/B Sequence Builder
Allow multiple sequence variants per campaign. The engine randomly (or weighted) assigns each lead to a variant. Track per-variant performance for A/B testing subject lines, opening hooks, and CTAs.
⏱ Weeks 5–7 · ~45h full-stack work
💡
Currently, each campaign has exactly one Sequence. This phase adds a SequenceVariant layer: a campaign can have 2–5 named variants (e.g. "Variant A — Pain-led", "Variant B — Social Proof"). Each variant has its own sequence steps. The engine assigns leads to variants at first contact and sticks to that variant for all follow-ups.
Database Changes
🗄
Add SequenceVariant Table + Lead Variant Assignment
Rename the current Sequence as the "default variant" (variant_index=0). Add a variant_name and weight field to Sequence. Add assigned_variant_id to Lead. Migration: set all existing leads' assigned_variant_id to their campaign's current (only) sequence ID.
ColumnTypeNotes
Sequence.variant_nameVARCHAR(100)e.g. "Variant A — Pain-led"
Sequence.weightFLOAT0.0–1.0, all variants sum to 1.0
Sequence.is_controlBOOLEANmarks the original/control variant
Lead.assigned_variant_idFK → sequences.idassigned at first contact; NULL = unassigned

Backend — Assignment Engine
⚙️
Weighted Random Variant Assignment on First Email
In campaign_engine.py, when scheduling a lead's first email (step 1), check if lead.assigned_variant_id is None. If so, pick a variant using weighted random selection based on each Sequence's weight field. Save the assignment to the lead record. All subsequent emails use the same variant_id. This ensures one lead sees only one consistent sequence of emails.
def assign_variant(campaign_id, lead, db): variants = db.query(Sequence).filter_by(campaign_id=campaign_id).all() weights = [v.weight for v in variants] chosen = random.choices(variants, weights=weights, k=1)[0] lead.assigned_variant_id = chosen.id db.commit() return chosen

05Phase
Live Campaign Editing with Auto-Adjustment
Allow full campaign settings editing after the campaign is active. Any change should intelligently cascade: reschedule pending emails, reallocate daily limits, reassign leads, rewrite sequences, and notify the user of what changed.
⏱ Weeks 10–13 · ~55h full-stack work
ℹ️
This is the most complex phase. The key insight: every campaign setting change is a "delta event" that must be processed by the campaign engine asynchronously. Changes don't take effect in the current sending window — they apply from the next Celery beat cycle (typically within 5 minutes).
Campaign Settings Panel (Always-Editable)
⚙️
Settings Tab — Full Edit Mode for Live Campaigns
Add a "Settings" tab to Campaign Detail (currently only editable pre-launch). All fields editable: name, daily limit, new leads/day, follow-up ratio, sending window, active days, timezone, tracking toggles. Each field shows its current value and has an Edit button. When editing, show a "Changes pending" banner. On Save, all changes go to a single PATCH endpoint.

Backend — Change Cascade Engine
🔄
CampaignChangeEvent Table + Async Cascade Processor
When a PATCH to /campaigns/{id} is received while campaign is ACTIVE, record a CampaignChangeEvent (change_type, old_value, new_value, applied_at=None). A new Celery task process_campaign_changes runs every 5 minutes and applies pending changes. Each change type has a specific cascade handler (see below).
Change TypeCascade ActionScope
daily_email_limit ↓Cancel excess ScheduledEmails for todayToday only
daily_email_limit ↑Allow more sends in next cycleNext cycle
sending_window changedReschedule all SCHEDULED emails outside new window to next valid slotAll future
active_days changedCancel emails on newly-excluded days; reschedule to next active dayAll future
timezone changedRecalculate all future scheduled_for timestamps in new timezoneAll future
sequence step editedApply to leads not yet reached that step; leave sent emails unchangedFuture steps only
sequence step deletedCancel any scheduled emails for that stepSCHEDULED only
track_opens toggledApply to all future sends (no change to past)Future sends

💡 Additional Suggestions Worth Considering
🤖
AI Email Generator
Add an "AI Generate" button in the sequence step editor. User provides ICP (Ideal Customer Profile) + value prop → Claude generates subject + body with merge tags already inserted. Include tone selector (Professional / Casual / Direct).
🏥
Email Warmup Module
Auto-manage inbox warmup for new accounts. Send warmup emails between your own accounts at increasing volume (5→10→20→50/day over 3 weeks). Track warmup progress in a visual curve. Pause campaign sending until warmup is complete.
🔗
CRM Sync (HubSpot / Pipedrive)
Webhook-based sync: when a lead replies and is marked INTERESTED or MEETING_BOOKED, auto-create a contact + deal in HubSpot/Pipedrive. Pull existing contacts to avoid emailing people already in your CRM. Two-way status sync.
📞
Reply Intent Classification
Use an LLM to classify incoming replies: Interested / Not Interested / Out of Office / Question / Meeting Booked. Auto-update lead status. Reduce manual triage. Confidence score shown — below 80% goes to manual review queue.
🕐
Time-Zone Aware Sending
Detect recipient's timezone from their email domain's MX geography or a MaxMind lookup. Schedule emails to arrive during 9–11am in the recipient's local timezone regardless of your sending window. Significantly improves open rates.
🧪
Spam Score Checker
Before saving a sequence step, run the email body through a spam scoring API (SpamAssassin or similar). Show a score 0–10 with highlighted phrases that are hurting deliverability: "free", "guarantee", all-caps words, excessive exclamations.
📂
Multi-Tenant / Team Support
Add an Organisation model. Users belong to an org. Campaigns, leads, and email accounts scoped to org. Role-based access: Owner / Manager / Viewer. Managers can't delete campaigns. Viewers are read-only. Essential for agency or team use.
📱
Mobile PWA with Push Notifications
Register the app as a PWA. Add a service worker. Send push notifications for: new reply detected, bounce spike (>5% in 1h), daily sending summary. The inbox especially benefits from mobile — quick status updates on the go.
